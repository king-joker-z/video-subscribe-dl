## 本轮目标（优先级排序）

### Task 1：Cookie 过期自动检测 + 降级通知 [P2]

**背景：** 当前 Cookie 过期后静默失败，用户完全不知道，体感极差（与今天修复的批量重试无反馈问题同类）。

**方案：**
- Cookie 验证失败时，后端设置全局状态 douyinCookieInvalid=true
- 新增 GET /api/douyin/cookie/status 接口（或复用已有）返回 cookie 状态
- dashboard 页面轮询该状态，Cookie 失效时展示醒目 banner
- Banner 上有"去设置"跳转按钮，点击跳转设置页
- Cookie 更新成功后 banner 自动消失（下次轮询时状态变为 valid）

**验收标准：**
1. Cookie 验证失败 → dashboard 出现醒目 banner
2. Banner 上有"去设置"跳转按钮
3. Cookie 更新成功后 banner 自动消失
4. 后端日志有对应的 WARN 级别日志

---

### Task 2：抖音下载进度追踪（SSE 集成）[P2]

**背景：** B 站已有完整的 SSE 进度推送（/api/events progress 事件），抖音下载静默进行，用户不知道进度。

**方案：**
- internal/scheduler/process_douyin.go 的 downloadDouyinFile 中，在下载循环里推送进度到 SSE channel
- 参考 B 站 dash_progress.go 的实现方式，复用已有的 SSE events 基础设施
- 前端 videos.js 已经有 SSE 订阅 + 进度条渲染（getProgress），后端只需要用同样的数据格式推送即可

**验收标准：**
1. 抖音视频下载时，前端视频列表出现进度条（与 B 站体验一致）
2. 显示速度 + 百分比 + ETA
3. 下载完成后进度条消失
4. 无内存泄漏（goroutine 正确退出）

---

### Task 3：前端 UX 小修 - 视频列表空状态优化 [P3-轻量]

**背景：** 筛选失败后空状态提示不够友好，没有"清除筛选"的快捷操作。

**方案：**
- EmptyState 组件增加可选的 action 按钮属性
- 当 status 或 search 有值时，空状态展示"清除筛选"按钮
- 点击后重置筛选条件

**验收标准：**
1. 有状态过滤时的空状态：显示"该状态下暂无视频 · 清除筛选"
2. 有搜索词时的空状态：显示"未找到匹配结果 · 清除搜索"
3. 无筛选时的空状态：原有提示不变
