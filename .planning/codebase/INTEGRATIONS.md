# External Integrations

## APIs & Services

### Bilibili (`api.bilibili.com`, `passport.bilibili.com`)

All calls use cookie/credential auth + WBI request signing (HMAC-MD5 over sorted params with mixin key).

| Endpoint | Purpose |
|---|---|
| `GET /x/space/wbi/acc/info` | Fetch UP main profile info (avatar, name) |
| `GET /x/space/wbi/arc/search` | List UP main uploaded videos (paginated) |
| `GET /x/web-interface/view?bvid=` | Get video detail (title, duration, owner, rights) |
| `GET /x/player/wbi/playurl` | Get DASH video+audio stream URLs (`fnval=4048`, up to 4K) |
| `GET /x/player/v2?bvid=&cid=` | Get subtitles list for a video |
| `GET /x/tag/archive/tags?bvid=` | Get video tags |
| `GET /x/polymer/web-dynamic/v1/feed/space` | Fetch UP main dynamics feed (incremental, up to 200 pages) |
| `GET /x/polymer/web-space/seasons_archives_list` | List season/collection videos |
| `GET /x/series/series?series_id=` | Get series metadata |
| `GET /x/series/archives` | List series videos (paginated, asc/desc) |
| `GET /x/v3/fav/folder/created/list-all` | List user's favourite folders |
| `GET /x/v3/fav/resource/list` | List videos in a favourite folder |
| `GET /x/v2/history/toview/web` | Get Watch Later list |
| `GET /x/relation/followings` | Get followed UP mains list |
| `GET /x/relation/followings/search` | Search followed UP mains by name |
| `POST /x/passport-login/web/cookie/refresh` | Refresh login cookie via `refresh_token` |
| `GET /x/passport-login/web/qrcode/generate` | Generate QR code for login |
| `POST /x/passport-login/web/qrcode/poll` | Poll QR code scan result |
| `GET https://passport.bilibili.com/x/passport-login/web/sso/list` | Post-login SSO list |
| `GET https://b23.tv/{short}` | Resolve Bilibili short URL (redirect follow) |

**WBI key source**: fetched from `https://www.bilibili.com` HTML page, extracting `img_url`/`sub_url` from nav data.

### Douyin (`www.douyin.com`, `www.iesdouyin.com`, `ttwid.bytedance.com`)

All calls require `X-Bogus` + `a_bogus` request signatures, generated by embedded JS (hot-reloadable from remote URL).

| Endpoint | Purpose |
|---|---|
| `GET /aweme/v1/web/aweme/detail/` | Get single video detail |
| `GET /aweme/v1/web/aweme/post/` | List user's posted videos |
| `GET /aweme/v1/web/user/profile/other/` | Get user profile |
| `GET /aweme/v1/web/mix/aweme/` | List collection videos |
| `GET /aweme/v1/web/query/user/` | Query user by unique ID |
| `GET /aweme/v1/web/aweme/favorite/` | Get user's liked videos |
| `GET https://www.iesdouyin.com/share/video/{id}` | Fallback video page scrape (no signing needed) |
| `GET https://www.iesdouyin.com/web/api/v2/aweme/slidesinfo/` | Image collection details |
| `POST https://ttwid.bytedance.com/ttwid/union/register/` | Register/obtain `ttwid` cookie token |

**Signing**: `sign.js` (X-Bogus) and `abogus.js` (a_bogus) run in a Goja VM pool; can be hot-reloaded from configurable remote URLs (checked every 6 hours via `sign_js_url` / `abogus_js_url` settings) with ETag-based caching.

### Pornhub (`www.pornhub.com`)

Scraped via HTML parsing (no official API). Uses `golang.org/x/net/html` parser.

| URL pattern | Purpose |
|---|---|
| `GET /{model-slug}` | Fetch model profile page (extract name from `<title>`) |
| `GET /{model-slug}/videos?page={n}` | Paginated video listing scrape |
| `GET /view_video.php?viewkey={id}` | Video detail page â€” extract `flashvars_*` JS variable for stream URLs |
| `GET /video/get_media?...` | Indirect API for MP4 direct links (requires login cookie) |
| HLS `.m3u8` URLs | Fallback stream URLs (240p/480p/720p/1080p) parsed from `mediaDefinitions` |

Proxy support: reads `HTTPS_PROXY` / `HTTP_PROXY` environment variables.

| Endpoint | Purpose |
|---|---|

Parsed via `compress/flate` + `encoding/xml`, optionally saved alongside video.

### Bilibili CDN (video/audio segments)

Direct DASH segment URLs returned by the playurl API. Host priority: `upos-*` (vendor CDN) > `cn-*` (self-hosted) > `mcdn` > `pcdn`. Large file downloads use a no-timeout `http.Client` with chunk-based download (50 MB threshold) and parallel chunks.

## Authentication

| Platform | Mechanism |
|---|---|
| **Bilibili** | Cookie-based session: `SESSDATA`, `bili_jct`, `buvid3`, `DedeUserID`, `ac_time_value` (refresh token). Stored as `credential_json` in SQLite `settings` table. Login via QR code scan (generate â†’ poll loop) or Netscape `cookie.txt` file import. `refresh_token` auto-refresh via `POST /passport-login/web/cookie/refresh`. |
| **Douyin** | Cookie string stored in SQLite `settings` table. Validated on update. Cookie expiry triggers `EventCookieExpired` notification. |
| **Pornhub** | Optional cookie string (enables higher quality MP4 via `video/get_media`; without it falls back to HLS). Stored in SQLite `settings`. |
| **App API** | Optional bearer token: `AUTH_TOKEN` env var. Token login endpoint: `POST /api/login/token`. Middleware checks `Authorization: Bearer` header or query param. |

## Data Sources

| Source | Type | Details |
|---|---|---|
| **SQLite database** | Local file | Single file at `{data-dir}/vsd.db` (path inferred from `glebarez/sqlite` init). Tables: `sources`, `downloads`, `settings`, `people`. |
| **Local filesystem** | File storage | Downloaded videos, thumbnails (`.jpg`), NFO metadata (`.nfo` XML), subtitles (`.srt`), danmaku (`.xml`/`.ass`) written to `{download-dir}`. |
| **Embedded static assets** | Go `embed.FS` | Compiled Vue SPA (`web/static/`), HTML templates, signing JS scripts bundled into binary. |
| **Remote signing JS** | HTTP fetch | `sign.js` and `abogus.js` optionally fetched from configurable URLs and hot-swapped in the Goja pool without restart. |

## Webhooks / Event Sources

### Outbound Notifications (app â†’ external)

The `notify` package dispatches events to one of three configurable channels (set via `notify_type` in settings):

| Channel | Endpoint / Config | Events |
|---|---|---|
| **Generic Webhook** | `POST {webhook_url}` â€” JSON payload `{event, title, message, timestamp}` | All event types |
| **Telegram Bot API** | `POST https://api.telegram.org/bot{token}/sendMessage` â€” MarkdownV2 format | All event types |
| **Bark (iOS push)** | `POST {bark_server}/push` (default: `https://api.day.app`) â€” JSON with `device_key`, `title`, `body`, `group`, `level`, `sound` | All event types |

**Event types**: `download_complete`, `download_failed`, `cookie_expired`, `disk_low`, `sync_complete`, `rate_limited`, `test`

All notifications are delivered via a buffered channel (capacity 64) with a single worker goroutine, retry up to 3 times with delays of 0s / 2s / 4s.
