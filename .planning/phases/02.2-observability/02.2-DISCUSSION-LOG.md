## Platform label on download counters

| Option | Description | Selected |
|--------|-------------|----------|
| Add Platform field to Job | Each scheduler sets job.Platform from source.Type; downloader increments per-platform map[string]*int64 atomically | ✓ |
| External counter callback on Job | OnComplete(platform, success) callback injected by schedulers; counter map lives outside downloader | |
| Counter map injected into MetricsHandler | Each scheduler maintains own counter; MetricsHandler reads via getter funcs; downloader unchanged | |

**User's choice:** Add Platform field to Job (Recommended)
**Notes:** Job.Platform set directly from source.Type string — no translation needed.

## Scheduler last-check wiring

| Option | Description | Selected |
|--------|-------------|----------|
| SetSchedulerLastCheckFunc per platform | Mirrors existing SetCooldownInfoFunc pattern; each scheduler exposes LastCheckAt() time.Time | ✓ |
| Single LastCheckFuncs map setter | SetSchedulerLastCheckFuncs(map[string]func() time.Time) — one setter call | |
| Direct struct refs via interface | SchedulerMetrics interface; MetricsHandler holds map[string]SchedulerMetrics | |

**User's choice:** SetSchedulerLastCheckFunc per platform (Recommended)

| Option | Description | Selected |
|--------|-------------|----------|
| Zero time.Time before first check | Return time.Time{} — Prometheus emits Unix 0 ("never") | ✓ |
| time.Now() at scheduler init | Initialize lastCheckAt to start time — less accurate | |

**User's choice:** Zero time.Time (Recommended)
**Notes:** Grafana/Prometheus users expected to handle counter resets and zero-value gauges with standard functions.
