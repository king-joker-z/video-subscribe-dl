### ✅ #9 — `MetricsHandler.getSchedulerLastCheck` is used in `HandlePrometheus` with `IsZero()` guard

**Evidence:**

`web/api/metrics.go` — struct field (line 19):
```go

```

Note: the plan spec initialised the map with `nil` values for each platform key; the actual
implementation uses an empty `make(...)` and populates entries via `SetSchedulerLastCheckFunc`.
This is functionally equivalent: a map key lookup for a missing key returns `nil` (the zero value
for `func() time.Time`), which is handled by the nil-function guard in `HandlePrometheus`.

`HandlePrometheus` (lines 160-168):
```go
for _, p := range platforms {
    ts := int64(0)
    if fn := h.getSchedulerLastCheck[p]; fn != nil {
        if t := fn(); !t.IsZero() {
            ts = t.Unix()
        }
    }
    fmt.Fprintf(w, "vsd_scheduler_last_check_timestamp{platform=%q} %d\n", p, ts)
}
```

Double guard: (a) nil function pointer, (b) `IsZero()` before calling `.Unix()`.
Zero-time guard prevents the infamous -62135596800 output. **PASS.**
