## Implementation Decisions

### P1-8：SSE completed 事件刷新方式
- **D-01:** 保留现有的 `setTimeout(load, 1000)` 整列表刷新方式，**不改为单条 `GET /api/videos/:id` 刷新**
  - 当前 `completed` 事件处理：先 patch 本地状态（status/file_size/downloaded_at），再延迟 1s 完整刷新列表
  - `[FIXED: P1-5]` 注释中的 `setTimeout(load, 1000)` 实质上已覆盖 P1-8 的需求（file_path、detail_status 等字段均可同步）
  - 补充 `// [FIXED: P1-8]` 注释，明确标注 P1-8 已被该方式解决
  - 理由：整页刷新语义更清晰，无需手动合并单条 API 返回；视频列表 20 条/页，开销可忽略
